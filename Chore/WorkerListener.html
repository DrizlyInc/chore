    class Chore::WorkerListener

    new(parent,timeout) click to toggle source

    The WorkerListener is a particular implementation of the PipeListener. It’s modeled after NewRelic’s method of shuttling data between processes. We use it for a similar purpose. It’s primarily used to transfer data from child processes to the master for tracking stats on the internal stat server.

  super(timeout)
  @parent = parent

    handle_payload(payload) click to toggle source

    handle_payload is called in the master process for each message that comes across the pipe. It unmarshals the payload into a simple hash structure.

    { 'type' => 'stat', 'value' => [key,val] }

    It then takes that data and passes it along to Chore.stats.add.

    # File lib/chore/strategies/worker/forked_worker_strategy.rb, line 18
def handle_payload(payload)
  Chore.logger.debug { "StatListener#handle_payload : #{Base64.encode64(payload)}" }
  data = Marshal.load(payload)
  if data['type'] && data['type'] == 'stat'
    Chore.stats.add(data['value'][0],data['value'][1])
  elsif data['type'] && data['type'] == 'status'
    @parent.workers[data['value']['id']].status = data['value']['status']
  end
  data = nil
rescue => e
  Chore.logger.error { "Failed to unmarshal data from pipe: #{e.inspect} : #{Base64.encode64(payload)}" }
end
